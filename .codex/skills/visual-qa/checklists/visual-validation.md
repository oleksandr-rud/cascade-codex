# Visual Validation Checklist

- [ ] Expected source named.
- [ ] Desktop and mobile/narrow viewports included or explicitly skipped.
- [ ] Long content and dynamic states considered.
- [ ] Text overlap, clipping, and horizontal overflow checked.
- [ ] Primary action and scan hierarchy checked.
- [ ] Token/component consistency checked.
- [ ] Accessibility-adjacent visual issues routed to `accessibility-review`.
- [ ] Functional failures routed to `functional-qa`.
- [ ] Evidence avoids sensitive data.
