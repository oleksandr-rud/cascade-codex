# UX Flow Review Checklist

- [ ] Actor, job, entry point, and completion signal named.
- [ ] Owning product spec and scenarios checked when available.
- [ ] Current UI route/components inspected when reviewing implemented UI.
- [ ] Loading, empty, error, blocked, success, and mobile states checked.
- [ ] Feature-specific UX routed to product docs.
- [ ] Reusable component/token rule routed to `design-system`.
- [ ] Accessibility concerns routed to `accessibility-review`.
- [ ] Evidence plan named before implementation or closeout.
