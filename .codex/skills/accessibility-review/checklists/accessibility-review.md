# Accessibility Review Checklist

- [ ] Scope, states, and user role named.
- [ ] Native HTML/Radix/shadcn semantics preferred before custom ARIA.
- [ ] Keyboard path checked or planned.
- [ ] Focus visibility and focus order checked or planned.
- [ ] Accessible names/descriptions checked or planned.
- [ ] Text and non-text contrast considered.
- [ ] Target size and mobile use considered.
- [ ] Forms, errors, and status messages considered when present.
- [ ] Findings distinguish confirmed issue, likely risk, and missing evidence.
- [ ] Report avoids legal/compliance attestation.
